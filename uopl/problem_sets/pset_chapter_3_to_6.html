Chapter 6: Pointers

1. Concept: What is a pointer? [answer]

A pointer is a reference that contains the memory address of something else.

2. Code: Write the full increment_number example that modifies an integer through a pointer. [answer]

increment_number :: proc(num: ^int) {
	num^ += 1
}

number := 7
number_pointer := &number
increment_number(number_pointer)
fmt.println(number) // 8

3. Code: Fetch the address of number into number_pointer. [answer]

4. Code: Call increment_number without first storing the pointer in a separate variable. [answer]

5. Concept: What is the type of number_pointer? [answer]

`^int`, meaning pointer to integer.

6. Code: Print a pointer as an integer number. [answer]

7. Code: Write the shorter increment_number procedure shown later in the explanation. [answer]

	num^ += 1
}

8. Code: Expand num^ += 1 into the equivalent assignment. [answer]

num^ = num^ + 1

9. Concept: What does ^int mean when the ^ is to the left of the type name? [answer]

It denotes a pointer type, here “pointer to integer.”

10. Concept: What does num^ mean when the ^ is to the right of a pointer variable? [answer]

It dereferences the pointer, reading or writing the value at the address stored in the pointer.

18. Code: Write the full birthday example that increments a cat's age through a pointer. [answer]

		return

19. Concept: Why does process_cat_birthday(&my_cat) modify my_cat directly? [answer]

Because `&my_cat` passes the address of `my_cat`, so the procedure can modify the original struct through the pointer.

replace_cat :: proc(cat: ^Cat) {

		age = 6,

	}
}

25. Concept: Why is cat^ = { ... } needed instead of cat = { ... }? [answer]

`cat = { ... }` would try to change the pointer itself. `cat^ = { ... }` changes the struct value that the pointer points to.
