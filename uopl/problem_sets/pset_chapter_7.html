7 Procedures and scopes overview

Code: Write the procedure example that checks whether one integer is bigger than another. [answer]

is_bigger_than :: proc(number: int, compare_to: int) -> bool {
	return number > compare_to
}

Code: Call `is_bigger_than` with `5` and `2`, then store the return value. [answer]

result := is_bigger_than(5, 2)

Concept: In the example `is_bigger_than(5, 2)`, what are `number` and `compare_to`, and what are `5` and `2`? [answer]

number and compare_to are parameters.

5 and 2 are arguments.

Concept: What is the difference between a parameter and an argument? [answer]

A parameter is what a procedure accepts, such as number: int or compare_to: int.

An argument is the value sent into the procedure when calling it, such as 5 and 2 in is_bigger_than(5, 2).

7.1.1 Allocator parameter with default value

Code: Write the `clone` procedure example with default `allocator` and `loc` parameters. [answer]

clone :: proc(s: []byte, allocator := context.allocator, loc := #caller_location) -> []byte {
	c := make([]byte, len(s), allocator, loc)
	copy(c, s)
	return c[:len(s)]
}

Code: Call `bytes.clone` with a single argument because the allocator has a default value. [answer]

copy := bytes.clone(some_bytes)

Code: Write the default allocator parameter from the `clone` example. [answer]

allocator := context.allocator

Code: Write the default source-location parameter from the `clone` example. [answer]

loc := #caller_location

Concept: Why is `allocator := context.allocator` a common pattern? [answer]

It gives a sensible default, while making the user understand that they can provide their own allocator if they want.

Concept: What is `#caller_location` used for in the core collection? [answer]

It fetches the source code position of where the procedure was called, which helps give good error messages.

7.2 Multiple return values

Code: Write `divide_and_double` with two return values: an `f32` result and a `bool` success value. [answer]

divide_and_double :: proc(n: f32, d: f32) -> (f32, bool) {
	if d == 0 {
		return 0, false
	}

	return (n/d)*2, true
}

Code: Write the return type list for two return values: `f32` and `bool`. [answer]

(f32, bool)

Concept: Why are parentheses required around multiple return types? [answer]

The parentheses are required because not having them would create ambiguities while the compiler is parsing the code.

Concept: What does `divide_and_double` return when `d == 0`? [answer]

return 0, false

Concept: What does the second return value mean in `divide_and_double`? [answer]

false indicates there was an error.

true indicates that everything went OK.

Code: Store both return values from `divide_and_double(2, 4)` in new variables and print the result if `ok` is true. [answer]

res, ok := divide_and_double(2, 4)

if ok {
	fmt.println(res)
}

Code: Assign both return values to pre-existing variables. [answer]

res, ok = divide_and_double(2, 4)

Concept: Why does the pre-existing variable assignment use `=` instead of `:=`? [answer]

Use = when assigning to pre-existing variables.

You cannot mix creating some new variables and re-using some old ones. Either all are new or all are pre-existing.

Code: Write the example that completely skips handling the return values. [answer]

Code: Add the attribute that requires callers to handle return values. [answer]

}

Code: Write the call that becomes an error when `@require_results` is used. [answer]

Code: Write the invalid example that handles only one return value. [answer]

Code: Write the example that keeps the first return value and explicitly throws away the second. [answer]

Code: Write the example that uses `_` to silence an unused `fmt` import. [answer]

Code: Write the special `if` statement that keeps `res` and `ok` scoped inside the `if`. [answer]

Concept: What is a common Odin pattern for reporting simple errors from a procedure? [answer]

Returning a bool in the last return value is a common way to report errors in Odin.

7.3 Named return values

Code: Write the named-return version of `divide_and_double`. [answer]

divide_and_double :: proc(n: f32, d: f32) -> (res: f32, ok: bool) {
	if d == 0 {
		return
	}

	res = (n/d)*2
	ok = true
	return
}

Code: Write the named return value list from the example. [answer]

(res: f32, ok: bool)

Code: Write the naked return used when `d == 0`. [answer]

if d == 0 {
	return
}

Code: Write the old explicit return that the naked return replaces when `d == 0`. [answer]

if d == 0 {
	return 0, false
}

Code: Write the assignment-based return path used when `d` is non-zero. [answer]

res = (n/d)*2
ok = true
return

Code: Write the old explicit return for the non-zero case. [answer]

return (n/d)*2, true

Concept: What is a naked return? [answer]

A naked return is a return that does not specify what it is returning.

It returns the named return values.

Concept: What values do `res` and `ok` have by default when `d == 0`? [answer]

res and ok are zero-initialized.

res contains 0.

ok contains false.

7.4 Nested procedures and captured variables

Code: Write the nested procedure example where `print_message` is declared inside `do_stuff`. [answer]

do_stuff :: proc() {
	print_message :: proc(msg: string) {
		fmt.println(msg)
	}

	print_message("Hellope!")
}

Code: Write the extended example with a global variable, global constant, local variable, and nested procedure. [answer]

global_state: int
CONSTANT_NUMBER :: 7

do_stuff :: proc() {
	local_variable: int

	print_message :: proc(msg: string) {
		fmt.println(msg)
	}

	print_message("Hellope!")
}

Concept: Which things can `print_message` use in the extended example? [answer]

print_message can use:

- The parameter msg
- The global variable global_state
- The global constant CONSTANT_NUMBER

Concept: Which variable can the nested `print_message` not see? [answer]

print_message cannot see or use local_variable that lives in the parent procedure do_stuff.

Concept: Why can nested procedures always use globals? [answer]

Globals live for as long as the program runs.

They are not bound to the lifetime of any procedure.

Code: Write the example of a static variable inside a procedure. [answer]

@static my_variable: int

Concept: How does a static variable inside a procedure behave? [answer]

A static variable behaves like a global variable, but it is only available within the procedure where it was declared.

Its value lives on between different calls to that procedure.

7.5 Parameters are always immutable

Code: Write the example that does not compile because it tries to change the parameter `n`. [answer]

divide_numbers :: proc(n, d: f32) -> f32 {
	n = n / d
	return n
}

Concept: What does immutable mean? [answer]

"mutable" means possible to change.

"immutable" means not possible to change.

Concept: What is true about all procedure parameters in Odin? [answer]

All procedure parameters are always immutable.

Code: Write the fixed version that creates a mutable local copy of `n`. [answer]

divide_numbers :: proc(n, d: f32) -> f32 {
	n := n
	n = n / d
	return n
}

Concept: What is shadowing in this example? [answer]

Creating a new variable with the same name as a procedure parameter is known as shadowing.

Code: Write the pointer-parameter procedure that modifies the value the pointer refers to. [answer]

increase_number :: proc(n: ^int, amount: int) {
	n^ += amount
}

Code: Write the example that does not compile because it tries to change what pointer parameter `n` points to. [answer]

increase_number :: proc(n: ^int, amount: int) {
	some_other_int := 7
	n = &some_other_int
	n^ += amount
}

Concept: For a pointer parameter, what is immutable: the address or the pointed-to memory? [answer]

The address that the pointer contains is immutable.

The memory it points to can still be modified.

7.6 Don't use pointer parameters for the sake of optimization

Code: Write the `Person` struct example. [answer]

Person :: struct {
	name: string,
	health: int,
	age: int,
	number_of_teeth: int,
	height_meters: f32,
}

Code: Write `print_person_info` using a value parameter `p: Person`. [answer]

print_person_info :: proc(p: Person) {
	fmt.printfln("%v is %v years old and has %v teeth.", p.name, p.age, p.number_of_teeth)
}

Code: Write the pointer-parameter version of `print_person_info`. [answer]

print_person_info :: proc(p: ^Person) {
	fmt.printfln("%v is %v years old and has %v teeth.", p.name, p.age, p.number_of_teeth)
}

Code: Show the pointer parameter form and the value parameter form for `Person`. [answer]

p: ^Person

p: Person

Concept: Why might `p: ^Person` seem reasonable at first? [answer]

Person uses 48 bytes of memory.

A pointer only uses 8 bytes on a 64 bit computer.

So it may seem like there would be way less copying if you used a pointer.

Concept: Why is `p: ^Person` not necessary just for optimization in Odin? [answer]

Because the size of Person is larger than 16 bytes, it will automatically be passed as an immutable reference.

Code: Write the example that checks how many bytes a type uses. [answer]

size_of(Person)

Code: Write the full `hans` example and pass it to `print_person_info`. [answer]

print_person_info :: proc(p: Person) {
	fmt.printfln("%v is %v years old and has %v teeth.", p.name, p.age, p.number_of_teeth)
}

hans := Person {
	name = "Hans",
	age = 65,
	number_of_teeth = 6,
	health = 20,
	height_meters = 1.69,
}

print_person_info(hans)

Concept: When `print_person_info(hans)` is called, what happens under the surface? [answer]

hans outside the procedure uses the exact same memory as p inside the procedure.

Under the surface the compiler has passed it as a reference.

However, the reference is immutable.

Code: Write the line that creates a modifiable local copy of `p`. [answer]

p := p

Concept: When should you use a pointer parameter? [answer]

Use a pointer parameter if you want to actually make a procedure that modifies something.

7.6.1 Pointer and immutable reference aliasing

Code: Write the `Person` and `Couple_Data` structs from the aliasing example. [answer]

Person :: struct {
	name: string,
	health: int,
	age: int,
	number_of_teeth: int,
	height_meters: f32,
}

Couple_Data :: struct {
	person_1: Person,
	person_2: Person,
}

Code: Write the `couple_data` initialization example. [answer]

couple_data := Couple_Data {
	person_1 = {
		name = "Hans",
		age = 65,
	},
	person_2 = {
		name = "Maj-Britt",
		age = 50,
	},
}

Code: Write the aliasing example where modifying `cd.person_1` also modifies `person`. [answer]

aliasing_example :: proc(cd: ^Couple_Data, person: Person) {
	cd.person_1 = {
		name = "This modifies 'person' too",
	}

	fmt.println(person.name) // Prints "This modifies 'person' too"
}

aliasing_example(&couple_data, couple_data.person_1)

Code: Write only the call to `aliasing_example`. [answer]

aliasing_example(&couple_data, couple_data.person_1)

Concept: Why does changing `cd.person_1` also change what `person.name` prints? [answer]

Because Person uses 48 bytes of memory, it is automatically passed as an immutable reference.

Inside aliasing_example, person is an immutable reference to the exact same memory as cd.person_1.

Code: Write the fixed aliasing example that makes an explicit copy of `person`. [answer]

aliasing_example :: proc(cd: ^Couple_Data, person: Person) {
	person := person

	cd.person_1 = {
		name = "This no longer modifies 'person'",
	}

	fmt.println(person.name) // Prints the original name
}

Concept: What does pointer aliasing mean? [answer]

Pointer aliasing means that you have two pointers that somehow reference the same data.

Changing data through one pointer alters what you see through the other pointer.

7.7 Explicit overloading

Code: Write the explicit overload group named `length`. [answer]

length :: proc {
	length_float2,
	length_float3,
}

Code: Write the 2D vector length procedure. [answer]

length_float2 :: proc(v: [2]f32) -> f32 {
	return math.sqrt(v.x*v.x + v.y*v.y)
}

Code: Write the 3D vector length procedure. [answer]

length_float3 :: proc(v: [3]f32) -> f32 {
	return math.sqrt(v.x*v.x + v.y*v.y + v.z*v.z)
}

Concept: What does `length :: proc { ... }` do? [answer]

It makes an explicit overload.

Inside the curly braces, you list all the procedures you want to be available under the name length.

Code: Use `length` with both a `[2]f32` and `[3]f32` value. [answer]

v2 := [2]f32 { 3, 3 }

len_v2 := length(v2)
len_v3 := length(v3)

Code: Call the specific overload variant `length_float2` directly. [answer]

v2 := [2]f32 { 3, 3 }
len_v2 := length_float2(v2)

Concept: What is a positive thing about explicit overloading? [answer]

You can still use a specific variant if you want.

Concept: Where does `math.sqrt` come from? [answer]

math.sqrt comes from:

import "core:math"

7.8 Run procedures on startup and shutdown

Code: Write the full `@init` and `@fini` example. [answer]

package init_fini_example

import "base:runtime"
import "core:fmt"

main :: proc() {
	fmt.println("Program running")
}

@init
startup :: proc "contextless" () {
	context = runtime.default_context()
	fmt.println("Program started")
}

@fini
shutdown :: proc "contextless" () {
	context = runtime.default_context()
	fmt.println("Program shutting down")
}

Concept: What does the `@init` and `@fini` program print? [answer]

Program started
Program running
Program shutting down

Concept: What does `@init` do? [answer]

Put @init in front of a procedure to automatically run it when your program starts.

Concept: What does `@fini` do? [answer]

Put @fini in front of a procedure to automatically run it when your program shuts down.

Code: Write the line used inside `startup` and `shutdown` to give `context` a value. [answer]

context = runtime.default_context()

Concept: Why are `startup` and `shutdown` marked `contextless`? [answer]

contextless is required for any procedure that has the @init or @fini attribute.

Concept: What is a calling convention? [answer]

A calling convention controls how a procedure is called, such as how it is passed parameters and how it returns things.

Concept: Why does the example set `context = runtime.default_context()` before calling `fmt.println`? [answer]

fmt.println expects to be implicitly passed a context.

Because startup and shutdown are contextless, the code first gives context a value.

7.9 The scope and the stack

Code: Write the simple procedure scope example with `number := 7`. [answer]

my_proc :: proc() {
	number := 7
}

Concept: Where is `number` valid in the simple scope example? [answer]

number is only valid within the scope of the procedure.

Code: Write the classic mistake: returning a pointer to a local variable through an intermediate variable. [answer]

my_proc :: proc() -> ^int {
	number := 7
	number_pointer := &number
	return number_pointer
}

Concept: Why is returning `number_pointer` a mistake? [answer]

The pointer is the address of the local variable number.

Once the procedure is over, number is no longer valid, so the pointer points at garbage memory.

Code: Write the return form that the compiler would reject directly. [answer]

return &number

Concept: What is a stack frame? [answer]

A stack frame is memory big enough to hold the procedure's local variables, parameters, and information about where to store the return values.

Concept: How much memory does an `int` use on a 64 bit system? [answer]

int is a 64 bit integer on 64 bit systems.

64 bits is equivalent to 8 bytes.

Concept: What happens to local variables when a procedure finishes? [answer]

When a procedure finishes, its stack frame is destroyed, or popped off the top of the stack.

Since local variables live inside the stack frame, there is no need to deallocate them manually.

7.10 Scopes within scopes

Code: Write the `if`-statement scope example where `message` is only available inside the `if` block. [answer]

my_proc :: proc() {
	number := 7

	if number > 5 {
		message := "It's more than 5"
		fmt.println(message)
	}

	// message is not available here
}

Concept: Which blocks create their own scope? [answer]

The code within the curly braces of any if, for, or proc has its own scope.

Concept: Which statement does not have its own scope? [answer]

The when statement does not have its own scope.

Code: Write the anonymous scope example. [answer]

my_proc :: proc() {
	number := 7

	{
		message := "Hellope"
		fmt.println(message)
	}

	// message is not available here
}

Concept: What is an anonymous scope? [answer]

An anonymous scope is code contained between two curly braces.

Concept: Why should you not use a pointer to `message` outside the anonymous scope? [answer]

The compiler backend might do optimizations where it reuses stack memory.

So you should never try to use such a pointer to save stack memory space.

Concept: When can anonymous scopes be useful? [answer]

Anonymous scopes can be useful in long procedures where you would otherwise get variable name collisions.

7.11 defer: Making things happen at end of scope

Code: Write the `read_file` example that uses `defer` to close a file handle. [answer]

read_file :: proc() {
	f, err := os.open("my_file.txt")

	if err != os.ERROR_NONE {
		// handle error
	}

	defer os.close(f)

	// Put code here that uses `f`
	// to read data from file.

	// os.close(f) is run at the end of
	// the procedure.
}

Code: Write the exact deferred close line from the file example. [answer]

defer os.close(f)

Concept: What does `defer os.close(f)` do? [answer]

It makes os.close(f) run at the end of the scope.

Concept: If `defer` is inside an anonymous scope or `if` block, when does it execute? [answer]

It executes at the end of that scope, not at the end of the procedure.

Code: Write the `most_green_color` example that unloads an image with `defer`. [answer]

most_green_color :: proc(filename: cstring) -> rl.Color {
	img := rl.LoadImage(filename)

	if img.data == nil || img.width == 0 || img.height == 0 {
		return {}
	}

	defer rl.UnloadImage(img)

	most_green_color := rl.GetImageColor(img, 0, 0)

	for x in 0..<img.width {
		for y in 0..<img.height {
			color := rl.GetImageColor(img, i32(x), i32(y))

			if color.g == 255 {
				// rl.UnloadImage(img) happens after this line
				return color 
			}

			if color.g > most_green_color.g {
				most_green_color = color
			}
		}
	}

	return most_green_color
} // <-- rl.UnloadImage(img) happens here.

Code: Write the deferred image cleanup line from `most_green_color`. [answer]

defer rl.UnloadImage(img)

Code: Write the early return section where the deferred unload still happens. [answer]

if color.g == 255 {
	// rl.UnloadImage(img) happens after this line 
	return color
}

Code: Write the final return section where the deferred unload also happens. [answer]

return most_green_color
} // <-- rl.UnloadImage(img) happens here.

Concept: What is powerful about `defer` when a procedure has multiple returns? [answer]

You can defer something once, but have it happen in different places depending on how the scope ends.

This is useful when returning early from procedures.

Concept: If there are multiple defers within a scope, what order do they run in? [answer]

They happen in the reverse order of how they were created.

7.11.1 Don't overuse defer

Concept: Why can `defer` reduce bugs? [answer]

It reduces bugs related to having return in multiple places.

Concept: Why can overusing `defer` make code harder to read? [answer]

You can't read the code in a serial manner anymore.

Concept: When does the author personally use `defer`? [answer]

The author uses defer in procedures like most_green_color, where cleanup code should run regardless of how the procedure ends.
