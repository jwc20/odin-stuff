8 Fixed-memory containers

1. Concept: What is a container in this chapter? [answer]

Containers are things in your program that can contain several items of a certain type.

2. Concept: Why are fixed arrays, enumerated arrays, and FCD Arrays called fixed-memory containers? [answer]

They are simple containers that do not require manual memory management and cannot grow larger than the initial size we give them.

3. Concept: What does FCD Array stand for? [answer]

Fixed Capacity Dynamic Array.

Bugs where you accidentally use index i-1 or i+1 instead of i are very common and are referred to as off-by-one errors.

16. Concept: In `for v, i in some_ints`, what does the `, i` add? [answer]

It adds another loop variable for the index.

17. Concept: Is `v` mutable in a normal `for v in some_ints` loop? [answer]

v is immutable, meaning it is not possible to modify.

18. Code: Loop through `some_ints` and multiply each element by 2. [answer]

for &v in some_ints {
	v *= 2
}

19. Concept: In `for &v in some_ints`, does `&v` give you a pointer? [answer]

The & here does not give you a pointer to the element. It gives you an addressable that you can assign to.

20. Concept: Why might looping without `&` be slow for arrays of big structs? [answer]

Without the &, each lap of the loop makes a copy of the element. This can be of concern if you iterate an array of big structs.

8.1.1 Where does the memory live?

23. Code: Inside `main`, create a fixed array of 100 integers. [answer]

main :: proc() {
	some_ints: [100]int
}

24. Concept: Where does a fixed array created inside a procedure live? [answer]

The memory of some_ints is part of the stack frame of the procedure main.

25. Code: Show that assigning a fixed array to a new variable copies all array data. [answer]

main :: proc() {
	some_ints: [100]int
	some_ints[10] = 2
	some_ints2 := some_ints
	some_ints2[10] = 5
	fmt.println(some_ints[10]) // 2
	fmt.println(some_ints2[10]) // 5
}

26. Concept: What does the copy example prove about `some_ints` and `some_ints2`? [answer]

This shows that there is no shared memory between these two fixed arrays.

27. Code: Give the two ways from the text to get a pointer to the first element of a fixed array. [answer]

raw_data(&some_ints)
&some_ints[0]

28. Code: Define a struct containing a fixed array of 100 integers. [answer]

My_Struct :: struct {
	some_ints: [100]int,
}

29. Code: Show that copying a struct copies the fixed array inside it. [answer]

main :: proc() {
	a_struct: My_Struct
	a_struct.some_ints[10] = 2
	another_struct := a_struct
	another_struct.some_ints[10] = 5
	fmt.println(a_struct.some_ints[10]) // 2
	fmt.println(another_struct.some_ints[10]) // 5
}

30. Concept: What does the struct-copy example show about fixed arrays inside structs? [answer]

Fixed arrays take up space directly within the struct. If you create a copy of a struct, then you get a copy of all the fields it contains, including fixed arrays.

31. Concept: What happens if your struct has a pointer field and you copy the struct? [answer]

If your struct has any kind of pointer, then the data which the pointer refers to will not get copied.

32. Concept: What is a shallow copy according to this section? [answer]

Only data that directly takes up space within the struct will get copied.

8.1.2 Blowing up the stack

33. Code: Show a giant fixed array inside `main` that may run out of stack memory. [answer]

main :: proc() {
	lots_of_ints: [1000000]int
}

34. Concept: Why can a giant fixed array inside a procedure crash your program? [answer]

Since the stack frame of each procedure has a limited amount of memory available, making a giant fixed array is a common way to run out of stack memory.

35. Code: Show a large global fixed array that is not part of a procedure's stack memory. [answer]

lots_of_ints: [10000000]int

main :: proc() {
	// lots_of_ints is a global variable
	// that you can use for as long as your
	// program runs.
}

36. Concept: Where do global variables live instead of the stack? [answer]

Global variables live inside what is known as the data block of the program.

37. Concept: What does the text recommend if you really need a fixed array that big? [answer]

If you really need a fixed array that big, then you are probably better off dynamically allocating memory.

8.1.3 Vectors and array programming

38. Code: Declare a 3D position using a fixed array of three `f32` values. [answer]

41. Code: Declare and assign a 3D position in one line. [answer]

position := [3]f32 {7, 1, 2}

42. Concept: For arrays with 4 or fewer items, what can you write instead of `position[0]`? [answer]

When arrays have 4 or fewer items, then you can also index them using x, y, z and w. So instead of position[0] you can write position.x.

43. Code: Create a new `[2]f32` from the z and x parts of `position`. [answer]

zx_pos := position.zx

44. Concept: What is swizzling? [answer]

You can create a smaller array from a bigger one using xyzw letters. You can also do stuff like position.xxyy or position.zyx. These kinds of operations are known as swizzling.

45. Concept: What other letters can you use instead of `.xyzw` for color-style vectors? [answer]

Because some applications use 4 dimensional numbers to represent colors, you can also use the letters .rgba instead of .xyzw.

46. Code: Create a type alias named `Vector3` for `[3]f32`. [answer]

47. Concept: Why is the name `Vector3` used? [answer]

Because Vector3 means 3D Vector. A 3D vector is a direction plus a length in 3D space, which is representable using 3 floating point numbers.

50. Concept: What is array programming in this context? [answer]

As long as two fixed arrays have the same element type and the same number of elements, then they can be added.

54. Concept: What is a scalar? [answer]

A single number is known as a scalar. This scalar must be of the same type as the fixed array's elements.

55. Concept: Does Odin have operator overloading? [answer]

Odin does not have operator overloading.

56. Concept: Why does the author not miss operator overloading much for vector math? [answer]

One of the most common use cases for operator overloading is doing vector maths. The author has not missed operator overloading because of how great array programming works.

It lives as a global in the data block of your program.

It would have the same number of items as the enum, but all of them would be zeroed.

You have to write #partial when you do a partial initialization. Either you mention all enum members, none, or use #partial.

Enumerated arrays work just like fixed arrays. They live on the stack and copy in the same way. No extra memory allocations happen.

8.3 Fixed Capacity Dynamic Arrays

72. Concept: What does an FCD Array behave like, even though it cannot grow past its initial capacity? [answer]

It behaves as if it can grow, even though it cannot grow past its initial capacity.

73. Concept: What type did the FCD Array replace? [answer]

Before the FCD Array existed in Odin, there was a type in core called Small_Array. The FCD Array replaced Small_Array.

74. Concept: What is the syntax difference between a dynamic array and an FCD Array? [answer]

Dynamic arrays use [dynamic]T syntax. FCD Arrays use [dynamic; N]T syntax.

75. Code: Create an FCD Array of integers with capacity 1024. [answer]

arr: [dynamic; 1024]int

76. Code: Write the full FCD Array example program from the text. [answer]

package fcd_array_example

import "core:fmt"

main :: proc() {
	arr: [dynamic; 1024]int

	append(&arr, 5)
	fmt.printfln("len: %v", len(arr))
	fmt.println(arr)

	append(&arr, 7)
	fmt.printfln("len: %v", len(arr))
	fmt.println(arr)

	append(&arr, 11)
	fmt.printfln("len: %v", len(arr))
	fmt.println(arr)

	// Remove element at index `0`. This
	// may reorder the other elements.
	unordered_remove(&arr, 0)
	fmt.printfln("len: %v", len(arr))
	fmt.println(arr)
}

77. Concept: What does `append` do in the FCD Array example? [answer]

It uses append to append things to the array.

78. Concept: What does `unordered_remove` do in the FCD Array example? [answer]

It uses unordered_remove to remove things from the array. unordered_remove may reorder the other elements.

79. Concept: What does `ordered_remove` do differently from `unordered_remove`? [answer]

ordered_remove preserves the order of the elements, but may be slower.

80. Code: Write the output of the FCD Array example program. [answer]

len: 2
[5, 7]
len: 3
[5, 7, 11]
len: 2
[11, 7]

81. Code: Show the runtime struct that stores FCD Array data. [answer]

Raw_Fixed_Capacity_Dynamic_Array :: struct($Capacity: uint, $T: typeid) {
	data: [Capacity]T,
	len:  int,
}

82. Concept: What two fields does the FCD Array contain under the hood? [answer]

A data field, which is a fixed array. A len field, which keeps track of how many elements of data are actually in use.

83. Concept: What happens when an FCD Array is full and `append` is called? [answer]

When it is full, append will no longer do anything. It will then return 0 to indicate that nothing was added to the array.

84. Concept: Where does the FCD Array live if it uses a fixed array under the hood? [answer]

Since it uses a fixed array, the FCD Array lives in stack memory.

85. Concept: What happens when you assign an FCD Array to a new variable? [answer]

Assigning an FCD Array to a new variable copies all its data.

86. Concept: How much stack memory does `[dynamic; 1024]int` use on a 64-bit system, according to the text? [answer]

The FCD Array with room for 1024 integers will use 1024*size_of(int) = 1024 * 8 = 8192 bytes of stack memory. It will actually use 8 bytes more since it also contains the len field: A total of 8200 bytes.

87. Concept: What is the main downside of an FCD Array compared with a dynamic array? [answer]

We can't start small and grow over time with an FCD Array. A dynamic array can start tiny and then grow as it runs out of space, but it requires manual memory management.
